# Healthcare Case Study (Anonymized)

The go-to case study for any healthcare prospect. Full text of the customer's story, with operator notes on when to use which parts.

## The client (anonymized)

- Leading not-for-profit healthcare organization
- Extensive network of primary care + outpatient services
- Hospitals + home care + specialized medical services
- Focus areas: cardiology, cancer, orthopedics, women's health
- Hospitals ranked among the best in the nation
- 13,000+ employees
- Serves 4 US regions
- One of the largest healthcare providers in the mid-Atlantic region

**Most likely real identity:** Based on size + region + service mix, this reads like WellStar or Christiana Care (both confirmed closed-wons). Confirm internally before suggesting similarity in outbound.

With Onyxia, we are able to save at least 160 working hours a month that one full-time analyst would traditionally spend on manual reporting and instead, focus more time on our actual risk management strategy."

## The pain (pre-Onyxia)

- Hours spent pulling reports from dozens of disconnected security tools
- Inefficient team time
- Point-in-time inaccuracy → lagging risk management + wasted budget
- No central source of truth for security stack efficiency + performance
- Needed better compliance with HIPAA and HITRUST
- Needed to proactively reduce breach/attack risk

## The Onyxia deployment

**Onboarding:** "Our client was able to quickly move from manual spreadsheets to a fully customized program performance dashboard. They integrated their entire security stack via APIs and were able to see the raw data turn into actionable intelligence *within an hour*."

**Time to value:** < 1 hour from integration to actionable intelligence.

## What changed (outcomes)

### 1. Program assessment & benchmarking
- Instantly select CPIs from the library
- Connect multiple resources per CPI
- See exact formula used to measure each CPI
- Adjust SLA per program needs
- Benchmark against industry SLAs
- Historical performance data over the past year

### 2. Streamlined board reporting
- Previously built from scratch manually
- Now: one-click PDF with TOC + granular CPI view + contextualized labels + definition + business value + performance over time
- Monthly + quarterly preset reports
- Better buy-in from stakeholders
- Validates security investment

### 3. Full security stack + asset coverage visibility
- Security Stack Map auto-generated on integration
- Identifies gaps + overlaps
- Ensures alignment with key frameworks
- Measures security tool coverage across all devices
- Cost-saving opportunities via overlap identification

### 4. HIPAA + HITRUST framework compliance dashboard
- One-click transform of the dashboard to HIPAA or HITRUST view
- Parent security categories + CPIs remap
- Security Stack Map automatically plots products into HIPAA/HITRUST pillars

### 5. OnyxAI predictive insights
- Statistical analysis + ML + NLP applied to the healthcare ecosystem data
- Suggestions for program improvement
- SLA optimization guidance
- Trend predictions to reduce risk + prevent future crises

## How to use this case study in outreach

### For healthcare-system prospects (FMOL, any hospital network):
Lead with 160 hours/month savings + HIPAA/HITRUST framework compliance one-click. Close with Health Infrastructure Security and Accountability Act as forcing function.

### For health tech / healthcare software (GenomOncology, Medusind):
Lead with integration time (< 1 hour) + Stack Map gap analysis. The "spreadsheet from hell to automation" quote from the insurtech CISO pairs well here.

### For recent-breach healthcare (Medusind — post $5M settlement):
Lead with board reporting + trend prediction. Frame as "preventing the next $5M settlement." Don't quote the settlement amount to the prospect. Let them connect the dots.

## Regulatory tailwinds to cite in healthcare outreach

- **Health Infrastructure Security and Accountability Act** (proposed) — mandates healthcare cybersecurity standards + enforcement for providers
- **HHS Cybersecurity Performance Goals (CPGs)** — voluntary but increasingly enforced
- **HIPAA Security Rule changes** (per 2026-02-13 internal Slack note, industry shift flagged by two GRC leaders)
- **HITRUST** — for healthcare orgs handling PHI at scale
